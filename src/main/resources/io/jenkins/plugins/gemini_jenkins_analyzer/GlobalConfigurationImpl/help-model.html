The Google Gemini model to use for error analysis and explanation.

Recommended models by use case:

Best for Most Users (Balanced Performance)

  • gemini-2.0-flash - Stable, fast, and cost-effective with excellent error analysis (recommended default)
  • gemini-2.0-flash-exp - Latest experimental Flash model with improved reasoning for advanced users
  • gemini-1.5-flash - Proven reliability for production environments
  • gemini-1.5-flash-002 - Latest stable 1.5 Flash variant with enhanced capabilities

High-Volume / Cost-Conscious

  • gemini-1.5-flash-8b - Fastest and cheapest, suitable for simple errors
  • gemini-2.0-flash-lite - Very fast with lower cost, good for frequent builds

Complex Errors / Advanced Analysis

  • gemini-1.5-pro - Most capable for complex multi-step debugging and analysis
  • gemini-1.5-pro-002 - Latest stable Pro variant with improved reasoning
  • gemini-2.0-pro-exp - Experimental Pro model with cutting-edge capabilities
  • gemini-thinking-exp-1219 - Extended thinking mode for particularly complex issues

Other Available Models

  • gemini-exp-1206 - Experimental snapshot with latest features
  • More models at Google AI Studio

Tips for Selecting a Model:

  • Start with gemini-2.0-flash (default) for the best balance of stability, speed, cost, and quality
  • Use Flash models (2.0-flash, 1.5-flash) for typical build failures, compilation errors, and test failures
  • Use Pro models (1.5-pro, 1.5-pro-002) for complex integration issues, obscure errors, or multi-service debugging
  • Use experimental models (*-exp) only if you want cutting-edge features and can tolerate potential API changes
  • Consider 8b or lite variants for high-volume CI/CD pipelines to reduce API costs
  • Monitor your API usage and costs; adjust model choice based on build frequency and complexity

Note: Model availability and performance may vary. Check the official documentation for the latest information.
