Number of log lines to include before and after each error line for context.

How it works:

  • When an error is detected, the preceding N lines are included
  • After an error, the following N lines are included
  • If no errors occur for N lines, that error block ends
  • Error lines are marked with >>> ERROR: prefix

Examples:

  • 3 (default): Good balance of context vs noise
  • 5-10: More context for complex build scripts
  • 0: Only error lines, no context

This helps the AI understand what led to the error and what happened immediately after.
