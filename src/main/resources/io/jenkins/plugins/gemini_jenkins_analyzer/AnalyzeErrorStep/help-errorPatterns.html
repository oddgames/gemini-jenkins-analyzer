Regular expressions to filter error logs (one per line). Only log lines matching these patterns will be sent to the AI for analysis.

Priority order:

  1. If specified here (in the pipeline step), these patterns will be used
  2. Otherwise, patterns from the job's Error Pattern Configuration will be used
  3. If none are specified, all logs will be analyzed without filtering

Example Unity patterns: 

(?i)\bError\s*:\s*
(?i)\bException\b
(?i)^\s*\*\*\s*BUILD FAILED\s*\*\*
(?i)error:\s*[^\s]

Patterns are case-insensitive by default when using (?i) flag.
